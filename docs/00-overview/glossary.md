# 术语表（Glossary）

## 1. 文档目的

本术语表用于统一 AetherTarot 项目中的核心词汇，减少产品、提示词、代码与评测之间的歧义。

---

## 2. 核心术语

### AetherTarot

项目代号。指面向深度塔罗占卜与反思式陪伴的智能体系统。

### Reading（解读）

指完整的一次塔罗输出结果，不仅包含逐牌解释，还包含综合主题、反思建议、安全提示与后续问题。

### Spread（牌阵）

指抽牌与位置关系的组合方式。牌阵定义包括卡牌数量、位置名称、每个位置含义、解释优先级与综合规则。

### Card Meaning（牌义）

指单张牌的基础含义，可能因正逆位、问题类型、上下文与牌阵位置而变化。

### Interpretation Framework（解释框架）

指系统如何把“问题 + 牌阵 + 单牌 + 关系 + 历史上下文”组合成完整解读的总规则。

### Reading Contract（解读契约）

指项目对用户承诺的服务边界，包括能提供什么、不能提供什么、如何表达不确定性。

### Long Context（长上下文）

指系统在长上下文环境下组织系统知识、用户输入、会话历史与持久化记忆的能力与策略。它的目标不是简单塞入更多文本，而是在上下文变长时仍保持问题贴合、结构稳定与安全边界清晰。

### Session Capsule（会话胶囊）

指从一轮会话中提炼出来的高密度摘要，用于后续会话延续，而不是简单保留全部历史原文。

### User Profile（用户画像）

指在明确授权和合规前提下沉淀的长期偏好信息，如语言风格偏好、关注主题、常见问题类型。

### Safety Boundary（安全边界）

指系统在敏感主题、危机话题、专业建议、关系操控等方面不可越过的限制。

### Hard Stop

指 Dual-Tier Safety 中的 Tier 1 升级处理。当系统识别到危机、自伤他伤、紧急医疗或重度操控等高风险意图时，后端直接中断解读生成并返回 `403` 安全拦截响应。

### Sober Check

指 Dual-Tier Safety 中的 Tier 2 降温审查。当用户试图把重大现实决策直接外包给塔罗时，系统仍返回 `200` 解读结果，但会强制加入需要用户先完成的现实反思摩擦。

### Output Schema（输出协议）

指解读结果的结构化字段定义，供前端、日志、评测与工具消费。

### `presentation_mode`

结构化 reading 中的呈现模式信标字段。它用于标记前端应采用的视觉与交互版式，并作为正式协议的一部分被记录与回放。

### `sober_check`

结构化 reading 中的安全字段。它在 Tier 2 决策外包场景下写入前端必须展示的降温审查问题，用于在阅读内容揭示前重新锚定用户自主性。

### Deterministic Claim（确定性断言）

指把象征性、启发性的塔罗解读包装成确定结果的表达方式，例如把趋势说成必然、把建议说成命令、把推断说成事实。该类表达被解读契约与安全原则明确限制。

### Eval（评测）

指用于判断输出质量与安全性的样例、评分规则、自动化检查与人工复核流程。

### ADR

Architecture Decision Record，架构决策记录。用于记录关键设计决策、原因、备选方案与后果。
