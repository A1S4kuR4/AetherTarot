## 1. 文档目的

本文件用于持续回答一个更具体的问题：

> 当前版本的 Web 主流程还剩下哪些尚未关闭的 UX / 产品风险？

当前最新评审结论（`2026-04-17`）是：**架构层已闭环，生产细节已打磨，回归信号已恢复；运行时牌池现已完整，且连续性边界与危机真实化已补上，当前主线风险重新收束到牌阵机制的持续扩展、长期连续性设计与文档持续同步。**

本文件作为持续更新的状态文档使用。当前记录显示：**三足鼎立空间架构 (ADR 0003)** 彻底解决了首页叙事与功能冲突的问题；同时在 `2026-04-12` 完成了 `/ritual` 路径的极致布局压缩（1080p 零滚动）、`/reveal` 路径的标签裁剪修复，并补上了 `/new -> /ritual -> /reveal` 的第一版焦点校准与“被组织的随机”解释。`2026-04-14` 的补充判断是：Playwright smoke 已重新对齐当前真实路径，回归信号恢复可用；`2026-04-15` 的补充判断是：运行时牌池已补齐到 `78/78`，且首个新增高价值运行时牌阵 `four-aspects` 已上线；`2026-04-17` 的补充判断是：hard-stop 已切到真实中国大陆危机 / 心理支持入口，continuity capsule 已新增高风险细节净化，且第二个高价值运行时牌阵 `seven-card` 已上线，因此 UX 主线的优先级进一步转向不同牌阵的前台差异化展开、结果理解机制与长期关系入口本身。

## 4. P0 剩余风险

### R1. 核心价值感不清

**当前判断：机制化关闭**

通过 **ADR 0003** 实施的全屏叙事快照滚屏，用户在进入功能前必须经过三段式的“意图校准”叙事（绪论、象征、心智）。这强制建立了“塔罗是内心的镜子”这一核心品牌心智。

**为什么已被解决**

- **叙事前置**：首页不再直接展示表单，而是通过物理滚屏节奏强行建立品牌深度。
- **空间解构**：仪式设置被移入独立的 `/new`（午夜模式），将“功能”与“发现”在视觉和路径上彻底解耦。
- **意图收束**：最后一屏的“抉择门户”将单纯的点击变成了一次基于意图的路径选择。

### R2. 抽卡被感知为纯随机程序噪声

**当前判断：进一步缓解，仍需观察**

这轮 UX 改动确实让抽牌不再像一次普通按钮点击。仪式页、洗牌动作、逐张抽取、揭示页都增强了体验层的成立感；`/new` 现在也已经补上了第一版焦点校准、“更适合问 / 尽量别问”示例，以及“你选择的是阅读容器，不是操纵结果”的解释。

`2026-04-20` 已进一步把“被组织的随机”从 `/new` 与 `/ritual` 的短提示推进到 `/reveal` 与 `/reading` 的可见机制：`/reveal` 解释本轮牌阵如何把随机牌面组织成阅读结构，`/reading` 在正式解读前说明逐牌顺序来自权威牌阵位置，先保留牌面线索与位置语义，再进入综合推断。

但从更强的机制证明层看，风险仍不应被视为完全关闭。

**为什么未被解决**

- 抽牌逻辑仍然是直接随机洗牌、随机抽取、随机逆位
- 当前焦点校准与组织解释仍主要停留在前台提示层，尚未进入更强的状态机或 schema 支撑
- 目前能解释“随机如何被牌阵阅读”，但仍不能也不应宣称“随机被用户意念操控”

**建议优先动作**

- 继续观察 `/reveal -> /reading` 的机制说明是否足够清楚
- 在不新增牌阵的前提下，继续打磨现有 5 个牌阵的前台展开重点
- 避免把“被组织的随机”说成决定论、意念操控或命运保证

### R3. 迎合错觉

**当前判断：部分缓解，仍需继续观察**

当前版本已经比“深背景输入 + 大段个性化输出”的形态更克制，因为前台只收一个问题输入。

`2026-04-19` 已在 reading 页补入第一版证据分层：逐牌展示显式拆为“牌面线索 / 位置语义 / 综合推断”，综合段也被标记为综合推断层。这让用户更容易看见哪些内容贴近牌面与牌阵，哪些内容属于解释整合。

但这不等于风险已经完全关闭。

**为什么仍未完全解决**

- 结果页已经开始区分“牌面线索”和“系统推断”，但层级节奏仍需继续观察真实使用反馈
- 综合段仍会直接围绕用户问题进行组织
- 当前前台已提供第一版分层，但还没有针对不同牌阵形成更差异化的证据展开方式

**建议优先动作**

- 继续打磨结果页“牌面线索 / 位置语义 / 综合推断”的节奏与视觉权重
- 控制对用户原话的复述密度
- 保留少量不完全顺着用户期待的张力点

### R4. 赛博塔罗感过强

**当前判断：显著缓解**

仪式页与揭示页已经明显降低了“直接出报告”的感觉。通过 `2026-04-12` 的布局优化，仪式感得到了物理层面的保障（无滚动干扰）。

**为什么已被缓解**

- **视口锁定**：仪式页现在能够完美适配 1080p 屏幕，无需滚动即可看到总结面板，增强了“神圣空间”的封闭感。
- **视觉修复**：修复了揭示页标签裁剪问题，并引入了 `REVERSED` 文字标识，让“赛博”呈现更显专业与严谨。
- **交互降噪**：侧边栏配色规范化，降低了不必要的视觉引导干扰。

但当用户进入最终解读页后，体验仍然很容易回到“结构化系统输出”的认知。

**为什么未被解决**

- reading 页依然是稳定字段分块
- 从 reveal 到 reading 的转折更多是视觉跳转，不是理解上的转折
- 不同牌阵在前台展开方式上的差异还不够大

**建议优先动作**

- 让 reveal 先给出整组牌的整体氛围和主张力
- 降低 reading 页的模块对称感
- 为不同牌阵设计不同的前台展开重心

## 5. P1 剩余风险

### R6. 用户把产品当作决策外包或情绪依赖入口

**当前判断：进一步机制化，但仍需继续观察**

当前已经引入 Dual-Tier Safety：危机 / 操控类问题会走 `403 Hard Stop`，重大现实决策外包会返回 `sober_check` 并在前端形成手写反思摩擦。`/new` 也已经前置加入了第一版提问校准、重大决策敏感词提醒，以及“可问 / 不适合问”的基础教育。

`2026-04-19` 进一步把重大现实决策类提问的前置提醒改成交互确认：用户必须先勾选确认“塔罗只用于整理线索，现实信息、专业意见与个人底线优先”，才能进入抽牌仪式。后续服务端 `sober_check` 仍会在 reading 结果揭示前继续形成手写现实反思摩擦。

同日还在 `/new` 增加了非阻断的重复主题提醒：当本地 completed history 中最近若干条记录与当前问题属于相同 `question_type` 时，前台提示用户先回看上一条线索，确认这次真正新增的问题是什么。该提醒只读取本地 history，不把历史注入 reading API，也不启用长期记忆。

这说明“重大决策类问题可完全正常进入流程”的风险已被进一步机制化处理，不再只依赖免责声明和后置 `safety_note`。

**为什么未被解决**

- 重复同类追问已有非阻断提醒，但还没有高频使用的冷却或节制机制
- 没有针对高频使用的节制机制
- 关系确认、情绪依赖和非关键词式决策外包仍可能进入正常流程
- 前置现实边界确认已覆盖明显重大决策词，但尚未覆盖非关键词式的情绪依赖与关系确认

**建议优先动作**

- 观察重复主题提醒是否能促使用户回看旧线索，而不是连续追问同一焦虑
- 继续扩展非关键词式决策外包与情绪依赖识别
- 观察前置确认是否降低用户把结果当成唯一决策依据的倾向

### R5. 留存不足，history 还不是复访理由

**当前判断：显著改善，部分闭环**

历史页已完成从“平铺列表”向“主题分群”的重构。通过**语义主题聚类（Theme Clustering）**与显式 continuity source，用户过去的记录不再只是孤立的噪声，而开始被收束为“可回看、可延续”的线索集合。

**为什么已被解决/缓解**

- **语义聚类**：`/journey` 默认提供“主题星群”模式，用户可以直观看到自己在“事业”、“情感”、“心理”等不同维度的回响深度。
- **复访动力**：每一组聚类都像一个“意识文件夹”，鼓励用户在后续遇到同类问题时，先回看过去的“星群”。
- **延续入口**：`/history` 与 `/journey` 现在都允许显式选择“延续这条线”，把上一轮 `session_capsule` 作为低优先级 continuity source 带回下一轮。
- **重复主题提醒**：`/new` 会基于本地 completed history 对相近 `question_type` 给出非阻断提醒，把“你刚刚问过类似主题”转化为回看旧线索的入口，而不是直接鼓励继续抽牌。
- **资产沉淀**：随着完整 `78/78` 运行时牌池与本地卡图闭环，历史记录的视觉与内容丰富度大幅提升。

**剩余风险**

- 手记仍主要挂靠在单次记录上，缺乏跨记录的“演化手记”功能。
- 本地存储限制了跨设备同步。

### R10. 结构化输出过稳，显模板化

**当前判断：部分缓解**

通过两阶段 reading、`presentation_mode` 呈现差异、以及基于牌面张力的追问交互，`InterpretationView` 已经不再完全等于“一次性吐出整份报告”。

**为什么已被缓解**

- **阶段差异**：Standard / Sober 会先给 initial reading，再通过牌面锚定追问进入 final，至少打破了“单回合定稿”的模板感。
- **安全摩擦**：`sober_check` 与 `presentation_mode = sober_anchor` 让一部分重大决策场景不再沿用普通 reading 的同质化节奏。
- **交互节奏**：追问输入与 final 收束至少让 reading 从静态回执，更靠近“先看牌、再校准、再整合”的过程。

**剩余风险**

- 最终深读（Final Reading）的布局仍受限于 Output Schema 的固定顺序。

### R11. 边界声明只在文档里强，产品内机制弱

**当前判断：部分缓解，产品内机制继续增强**

首页、侧边栏、结果页、`Hard Stop`、`Sober Check` 与 safety override 已经把边界搬进了产品。尤其是 `sober_check` 已经从后置文案升级为结果揭示前的交互摩擦，且 `/new` 现已提供“更适合问 / 尽量别问”的例句与第一版重大决策提醒。

`2026-04-19` 的前置现实边界确认把“这不是替你决定”从说明文案推进为进入抽牌前的用户动作。它不替代服务端 safety layer，但让普通用户在流程前半程就必须承认现实信息与个人底线优先。

**为什么未被解决**

- 边界仍有一部分停留在说明文案，尤其是普通敏感场景与长期使用行为
- 高风险中的 Tier 1 / Tier 2 已有机制，但普通敏感场景仍主要依靠 `safety_note`
- 产品前半程已有“可问什么 / 不适合问什么”例句与重大决策确认，但还需要覆盖更隐性的依赖型提问
- 重复追问已有本地提醒，但高频依赖与关系确认类问题还缺少持续性行为约束

**建议优先动作**

- 在提问前给出可问 / 不适合问的提示
- 对高风险问题在提交前做一次边界确认
- 更早地提供现实支持与专业帮助导向

### R7. 产品人格过于安全、温和、可预期

**当前判断：第一版机制化缓解，仍需观察**

UX 风格已经更有气氛，但生成逻辑层仍高度依赖固定问题类型镜头、固定 guidance 与固定追问模板。

`2026-04-20` 已在 provider 与 LLM prompt 约束中加入“建设性阻力”规则：reading 不应只顺着用户期待安抚，而要至少保留一个来自牌面、正逆位、位置语义、牌阵张力或现实未验证条件的观察点。后续读感校准已让关系、职业、自我成长、选择与综合议题使用不同阻力表达，避免把“建设性阻力”本身变成新模板。该观察点不能变成确定性预言、第三方读心、专业替代建议或命令式决策。

**为什么未被解决**

- 文本展开方式已开始保留张力点，但仍需观察长期使用是否足够有差异
- 长期使用时容易形成熟悉模式
- 安全与克制仍可能吞掉人格锐度，因此张力表达必须持续接受 safety 与 eval 约束

**建议优先动作**

- 继续观察不同问题类型下的建设性阻力是否真实可感
- guidance 不要每次都落回类似的安抚句式
- 在安全前提下保留少量“阻力感”，但不把阻力感升级为冒犯、断言或替用户做决定

### R8. 解释体系过强，用户价值感没有同步增强

**当前判断：仍然明显存在**

这轮 UX 主要改善的是界面与仪式感，并没有直接解决“用户为什么会持续回来”的核心价值问题。

**为什么未被解决**

- 当前体验仍主要围绕解释结构展开
- 用户长期价值尚未转化成产品机制
- 团队仍容易继续优化内部解释质量，而不是外部价值显性

**建议优先动作**

- 每次新增能力先问：它是否提升复访价值
- 优先做长期关系与阶段连续性，而不是继续堆内部完整度
- 减少只对内部结构有益、对用户体感无增益的展示

### R9. 页面流程完整，但节奏仍未完全成立

**当前判断：机制化关闭**

**ADR 0003** 引入的全屏快照滚动彻底重塑了首页节奏。

**为什么已被解决**

- **快照反馈**：滚轮不再是滑行，而是“帧对帧”的段落切换，建立了明确的叙事节奏感。
- **仪式跳转**：从首页叙事末尾跳转到独立仪式页（从纸张模式切换到午夜模式）形成了极强的体感冲突与心理仪式感。
- **路径分化**：`/new` 与 `/journey` 的独立化，让工具效率与情感沉淀互不干扰。

### R12. 本地持久化策略限制长期产品形态

**当前判断：未受本轮 UX 改动影响**

这轮 UX 改动没有改变持久化边界。当前历史与手记仍然依赖本地存储。

这意味着它仍然会在中期限制：

- 跨设备连续性
- 长期陪伴感
- 用户关系资产积累
- 自然演化为长期反思工具的能力

**建议优先动作**

- 在架构层尽早定义未来的长期连续性模型
- 即使短期不做服务端持久化，也要先定义演进边界

## 6. 当前建议

如果只按近期收益排序，下一轮 UX / 产品工作建议继续保持“校准读感”：

1. 先保持可信回归与状态同步
   继续把 `npm run test:e2e` 作为每次前台 UX 改动后的 P0 真相门槛，并让文档状态跟上真实产品路径。

2. 观察并微调 `R2 / R3 / R7`
   `R2` 已从通用“组织随机”推进到 5 个牌阵差异化机制说明；`R3` 已把证据路径显式放到 reading 机制区，但仍需观察用户是否真的理解“牌面线索 / 位置语义 / 综合推断”的层级；`R7` 已让 5 个牌阵都进入 provider / prompt 的结构轴线，后续继续抽样其张力是否有牌阵差异、不冒犯、不越界、不退回模板句。

3. 在上述读感稳定后，再推进 history / journey 的阶段性价值
   后续重点再转向 `R8 / R10 / R12`，但服务端 memory persistence、长期画像、memory merge 与新增运行时牌阵继续暂缓，直到 `R2 / R3 / R7` 的前台体验更稳。
