## 4. P0 剩余风险

### R1. 核心价值感不清

**当前判断：机制化关闭**

通过 **ADR 0003** 实施的全屏叙事快照滚屏，用户在进入功能前必须经过三段式的“意图校准”叙事（绪论、象征、心智）。这强制建立了“塔罗是内心的镜子”这一核心品牌心智。

**为什么已被解决**

- **叙事前置**：首页不再直接展示表单，而是通过物理滚屏节奏强行建立品牌深度。
- **空间解构**：仪式设置被移入独立的 `/new`（午夜模式），将“功能”与“发现”在视觉和路径上彻底解耦。
- **意图收束**：最后一屏的“抉择门户”将单纯的点击变成了一次基于意图的路径选择。

### R2. 抽卡被感知为纯随机程序噪声

**当前判断：部分缓解，仍然存在**

这轮 UX 改动确实让抽牌不再像一次普通按钮点击。仪式页、洗牌动作、逐张抽取、揭示页都增强了体验层的成立感；`2026-04-12` 又在 `/new` 补上了第一版焦点校准，并在 `/ritual` 与 `/reveal` 补上了“阅读容器如何组织随机”的解释。

但从用户理解层看，风险没有被真正关闭。

**为什么未被解决**

- 抽牌逻辑仍然是直接随机洗牌、随机抽取、随机逆位
- 当前焦点校准仍停留在前端提示层，尚未进入更强的状态机或 schema 支撑
- 没有机制证明“为什么这一轮随机与你的问题形成了这组映照”

**建议优先动作**

- 在抽牌前加入问题焦点校准
- 明确告诉用户：你选择的是阅读容器，不是操纵结果
- 在揭示页解释牌阵位置如何组织本次随机

### R3. 迎合错觉

**当前判断：仍然存在**

当前版本已经比“深背景输入 + 大段个性化输出”的形态更克制，因为前台只收一个问题输入。

但这并不等于风险已经消失。

**为什么未被解决**

- 结果页没有区分“牌面线索”和“系统推断”
- 综合段仍会直接围绕用户问题进行组织
- 当前前台没有任何地方帮助用户理解：哪些内容更接近牌阵支持，哪些内容属于解释扩展

**建议优先动作**

- 在结果页显式拆开“牌面线索 / 位置语义 / 综合推断”
- 控制对用户原话的复述密度
- 保留少量不完全顺着用户期待的张力点

### R4. 赛博塔罗感过强

**当前判断：显著缓解**

仪式页与揭示页已经明显降低了“直接出报告”的感觉。通过 `2026-04-12` 的布局优化，仪式感得到了物理层面的保障（无滚动干扰）。

**为什么已被缓解**

- **视口锁定**：仪式页现在能够完美适配 1080p 屏幕，无需滚动即可看到总结面板，增强了“神圣空间”的封闭感。
- **视觉修复**：修复了揭示页标签裁剪问题，并引入了 `REVERSED` 文字标识，让“赛博”呈现更显专业与严谨。
- **交互降噪**：侧边栏配色规范化，降低了不必要的视觉引导干扰。

但当用户进入最终解读页后，体验仍然很容易回到“结构化系统输出”的认知。

**为什么未被解决**

- reading 页依然是稳定字段分块
- 从 reveal 到 reading 的转折更多是视觉跳转，不是理解上的转折
- 不同牌阵在前台展开方式上的差异还不够大

**建议优先动作**

- 让 reveal 先给出整组牌的整体氛围和主张力
- 降低 reading 页的模块对称感
- 为不同牌阵设计不同的前台展开重心

## 5. P1 剩余风险

### R6. 用户把产品当作决策外包或情绪依赖入口

**当前判断：部分机制化，但仍然明显存在**

当前已经引入 Dual-Tier Safety：危机 / 操控类问题会走 `403 Hard Stop`，重大现实决策外包会返回 `sober_check` 并在前端形成手写反思摩擦。`2026-04-12` 又在 `/new` 前置加入了第一版提问校准与重大决策提醒。

这说明“重大决策类问题可完全正常进入流程”的风险已被部分机制化处理，不再只依赖免责声明和后置 `safety_note`。

**为什么未被解决**

- 没有针对重复同类追问的提醒
- 没有针对高频使用的节制机制
- 关系确认、情绪依赖和非关键词式决策外包仍可能进入正常流程
- `sober_check` 主要在结果揭示前形成摩擦，提问前的“可问 / 不适合问”教育仍不足

**建议优先动作**

- 对短时间重复同类问题加入节制提醒
- 对重大决策类问题加入“先补现实信息”的前置摩擦
- 把“这不是替你决定”从声明做成交互机制

### R5. 留存不足，history 还不是复访理由

**当前判断：显著改善，部分闭环**

历史页已完成从“平铺列表”向“主题分群”的重构。通过**语义主题聚类（Theme Clustering）**，用户过去的记录不再是孤立的噪声，而被收束为“生命议题星群”。

**为什么已被解决/缓解**

- **语义聚类**：`/journey` 默认提供“主题星群”模式，用户可以直观看到自己在“事业”、“情感”、“心理”等不同维度的回响深度。
- **复访动力**：每一组聚类都像一个“意识文件夹”，鼓励用户在后续遇到同类问题时，先回看过去的“星群”。
- **资产沉淀**：随着 78 张牌资源的逐步注入（当前已补齐圣杯 `Ace-10`），历史记录的视觉与内容丰富度大幅提升。

**剩余风险**

- 手记仍主要挂靠在单次记录上，缺乏跨记录的“演化手记”功能。
- 本地存储限制了跨设备同步。

### R10. 结构化输出过稳，显模板化

**当前判断：部分缓解**

通过重构 `InterpretationView` 引入了**“叙事桥梁（Narrative Bridge）”**机制。初读与深读之间的追问环节从“填表”变成了“回声校准”。

**为什么已被缓解**

- **节奏打破**：叙事桥梁用动态的对话感代替了死板的 Section 边框，让用户感到这不仅是系统生成，而是一次基于现实锚点的互动。
- **视觉去模板化**：移除沉重的 textarea 边框，改为信纸线条风格，降低了“填写调查问卷”的心理防御。
- **动态呼吸感**：逐条加载的 Motion 动效让追问具备了时间维度上的节奏，而不是一次性甩出三个表单。

**剩余风险**

- 最终深读（Final Reading）的布局仍受限于 Output Schema 的固定顺序。

### R11. 边界声明只在文档里强，产品内机制弱

**当前判断：部分缓解，仍然存在**

首页、侧边栏、结果页、`Hard Stop`、`Sober Check` 与 safety override 已经把边界搬进了产品。尤其是 `sober_check` 已经从后置文案升级为结果揭示前的交互摩擦，且 `/new` 现已新增提问前的第一版边界提示与重大决策确认。

这是正确方向，但还没有形成真正的用户行为引导。

**为什么未被解决**

- 边界仍有相当一部分停留在说明文案
- 高风险中的 Tier 1 / Tier 2 已有机制，但普通敏感场景仍主要依靠 `safety_note`
- 产品前半程虽然已有引导，但仍缺少更明确的“可问什么 / 不适合问什么”例句与反例
- 重复追问、高频依赖与关系确认类问题还缺少持续性行为约束

**建议优先动作**

- 在提问前给出可问 / 不适合问的提示
- 对高风险问题在提交前做一次边界确认
- 更早地提供现实支持与专业帮助导向

### R7. 产品人格过于安全、温和、可预期

**当前判断：仍然明显存在**

UX 风格已经更有气氛，但生成逻辑层仍高度依赖固定问题类型镜头、固定 guidance 与固定追问模板。

**为什么未被解决**

- 文本展开方式的变化幅度有限
- 长期使用时容易形成熟悉模式
- 安全与克制正在吞掉人格锐度

**建议优先动作**

- 让不同问题类型有不同语气张力
- guidance 不要每次都落回类似的安抚句式
- 在安全前提下保留少量“阻力感”

### R8. 解释体系过强，用户价值感没有同步增强

**当前判断：仍然明显存在**

这轮 UX 主要改善的是界面与仪式感，并没有直接解决“用户为什么会持续回来”的核心价值问题。

**为什么未被解决**

- 当前体验仍主要围绕解释结构展开
- 用户长期价值尚未转化成产品机制
- 团队仍容易继续优化内部解释质量，而不是外部价值显性

**建议优先动作**

- 每次新增能力先问：它是否提升复访价值
- 优先做长期关系与阶段连续性，而不是继续堆内部完整度
- 减少只对内部结构有益、对用户体感无增益的展示

### R9. 页面流程完整，但节奏仍未完全成立

**当前判断：机制化关闭**

**ADR 0003** 引入的全屏快照滚动彻底重塑了首页节奏。

**为什么已被解决**

- **快照反馈**：滚轮不再是滑行，而是“帧对帧”的段落切换，建立了明确的叙事节奏感。
- **仪式跳转**：从首页叙事末尾跳转到独立仪式页（从纸张模式切换到午夜模式）形成了极强的体感冲突与心理仪式感。
- **路径分化**：`/new` 与 `/journey` 的独立化，让工具效率与情感沉淀互不干扰。

### R12. 本地持久化策略限制长期产品形态

**当前判断：未受本轮 UX 改动影响**

这轮 UX 改动没有改变持久化边界。当前历史与手记仍然依赖本地存储。

这意味着它仍然会在中期限制：

- 跨设备连续性
- 长期陪伴感
- 用户关系资产积累
- 自然演化为长期反思工具的能力

**建议优先动作**

- 在架构层尽早定义未来的长期连续性模型
- 即使短期不做服务端持久化，也要先定义演进边界

## 6. 当前建议

如果只按近期收益排序，下一轮 UX / 产品工作建议优先聚焦三件事：

1. 先保持可信回归与状态同步
   重点要求 smoke 回归与文档状态继续跟上真实产品路径

2. 把首页与抽牌流程从“好看”推进到“可理解”
   重点解决 `R2` 与相关理解机制缺口

3. 把 reading 与 history 从“稳定报告 / 回放列表”推进到“有张力的阅读体验 / 阶段性自我记录”
   重点解决 `R3`、`R5`、`R10` 与 `R12`
