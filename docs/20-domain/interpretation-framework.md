# 解释框架（Interpretation Framework）

## 1. 文档目的

定义 AetherTarot 如何把用户输入、牌阵、单牌、上下文与风格规则组织成一次完整解读。

---

## 2. 输入组成

一轮解读的输入通常包括：

- 用户问题
- 牌阵类型
- 抽到的牌与正逆位
- 抽牌来源（线上随机或线下实体牌录入）
- 当前轮目标（`initial` / `final`）
- `agent_profile`（`lite` / `standard` / `sober`）
- 第一阶段快照与追问回答（仅 `final` 阶段）
- 上一轮 `prior_session_capsule`（如有，且仅作低优先级 continuity）
- 安全边界与输出协议

当前运行时补充：

- 前台首轮默认只收一个核心问题，不再默认收集大段自由背景。
- `standard / sober` 走两阶段：先让牌面建立主轴，再用追问回答缩小解释空间。
- `lite` 允许直接完成，不强制进入追问阶段。
- `prior_session_capsule` 只用于延续主题，不允许覆盖当前问题、当前牌阵与当前抽牌。
- 线下塔罗模式只改变 `drawnCards[]` 的来源：用户用实体牌自行抽取，前台按牌阵位置录入牌面与正逆位；后端仍以相同权威牌阵、相同安全边界和相同结构化输出协议生成解读。

---

## 3. 解读层级

建议按以下层级组织解释：

### 第一层：问题校准

先识别用户真正关心的问题是什么：

- 结果焦虑
- 关系确认
- 行动选择
- 情绪卡点
- 自我成长

当前实现补充：

- 服务层会先把问题归入 `relationship | career | self_growth | decision | other`。
- 这个分类是解释镜头，不是产品向用户宣告的“诊断标签”。
- 分类结果主要用于决定初读语气、追问重心与综合段落的组织角度。

### 第二层：逐牌理解

对每张牌进行基础解释，但必须考虑：

- 正逆位
- 牌所在位置
- 问题类型
- 与相邻牌的张力/呼应

### 第三层：牌阵关系

解释不同位置之间如何形成：

- 因果关系
- 过去-现在-未来路径
- 内外冲突
- 显性问题与隐性问题
- 阻力与资源

当前实现补充：

- 所有牌阵都必须先尊重位置语义，再做综合。
- 前台与 provider 都应明确“随机决定牌面与正逆位，牌阵决定阅读顺序、位置语义与综合路径”。这只能说明随机如何被组织成可阅读结构，不能宣称用户意念操控了结果。
- `/reveal` 与 `/reading` 当前已提供第一版“牌阵如何组织随机”说明，用于把抽牌体验从纯随机噪声推进到可复核的牌阵阅读。
- 当前明确写入运行时 bias 的牌阵是 `seven-card`：优先抓 `答案 / 当事人 -> 结果` 主轴，再回看 `过去 -> 现在 -> 最近结果` 的时间线，以及 `周遭能量` 与 `希望 / 恐惧` 的张力。
- 其余牌阵当前仍主要依赖通用位置语义与 provider 的逐牌-综合收束，不应假定它们已经拥有和 `seven-card` 同级别的显式编排规则。

### 第四层：综合主题

将逐牌解释上升为 2-4 个主题，例如：

- 失衡的付出与回收
- 被压抑的真实需求
- 关系中的边界模糊
- 行动时机未成熟但觉察已开始

### 第五层：反思建议

给出可执行但不命令式的建议，强调：

- 观察什么
- 可以如何提问自己
- 下一步适合做什么小行动
- 哪些决定不应仓促下结论

当前实现补充：

- `initial` 阶段的追问必须锚定牌面张力、位置语义或缺失的现实条件，而不是泛泛索取背景。
- `final` 阶段只能收束、校正、深化第一阶段主轴，不能把追问回答当成重写牌面的许可。
- `sober` profile 的建议与追问会更强调现实边界、风险承受与可验证条件。
- reading 应保留至少一个“建设性阻力”观察：它可以指出牌面、逆位、位置语义、牌阵张力或现实未验证条件并不完全支持用户期待的单一答案。
- 建设性阻力不能覆盖安全边界，不能变成确定性预言、第三方读心、专业替代建议或命令式决策。
- 不同问题类型的建设性阻力应有可感差异：关系类不能滑向第三方读心，职业类应回到现实反馈与资源约束，自我成长类应避免变成自责，选择类应强调未核实条件与代价。

---

## 4. 优先级规则

建议默认优先级：

1. 用户真实问题
2. 牌阵位置语义
3. 单牌语义
4. 牌与牌之间的关系
5. 历史上下文
6. 风格增强表达

当前补充约束：

- `initial_reading` 在 `final` 阶段优先级高于新的语言包装；第二阶段必须延续第一阶段主题。
- `prior_session_capsule` 低于当前问题、当前牌阵与当前抽牌，只能作为 continuity 背景。
- `agent_profile` 可以影响展开深度与现实校验强度，但不能覆盖牌阵逻辑或安全边界。

> 风格不能覆盖牌阵逻辑，更不能覆盖安全边界。

---

## 5. 正逆位处理原则

当前运行时约定：

- 当前运行时默认所有抽到的牌都允许出现逆位。
- 逆位不直接等于“坏”，而更接近力量方向、表达方式、成熟度或流动受阻的变化。
- 在关系、决策与成长类问题中，逆位通常更偏向提醒“哪里没有被好好表达、整合或落地”，而不是直接宣判结果恶化。
- 综合段应把逆位理解为节奏和关系的变化信号，而不是简单负面标签。

---

## 6. 常见失败模式

- 只做逐牌说明，没有整体综合
- 无视牌阵位置，只按固定牌义回答
- 过度神秘化，缺乏清晰结构
- 过度现实化，失去塔罗语言质感
- 把倾向说成结论
- 给出僵硬、空泛、套话式建议
- 追问脱离牌面，只是在套背景
- 第二阶段推翻第一阶段主轴
- 把 continuity 当成新的主问题，而不是低优先级延续线索
