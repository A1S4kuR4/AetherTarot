## 7. LLM 输出质量与 Follow-up 稳定性修复

### 7.1 触发背景
- 手动联调 `llama3.1:8b` 时，发现 provider 允许模型改写牌名 / 位置名，导致用户界面出现伪中文或错位标签。
- 同时发现当模型返回重复追问时，前端以追问文本本身作为 key，导致两个回答框共享同一份输入状态。

### 7.2 核心修复
- `apps/web/src/server/reading/llm-provider.ts`
  - `normalizeCards()` 改为锁定 authority card metadata：`name`、`english_name`、`position`、`position_meaning` 不再信任模型返回值。
  - 若 provider 返回的 `card_id`、`position_id`、`orientation` 与 authority 不一致，直接抛 `generation_failed`。
  - `normalizeStringArray()` 现在会先去重，再做长度裁剪与最小数量校验，避免重复 `follow_up_questions` 直接进入 UI。
- `packages/prompting/src/index.ts`
  - 明确要求所有用户可见文本必须是自然简体中文。
  - 明确禁止 `Thinking...`、思维链、分析痕迹和模型自述泄漏。
  - 明确禁止改写权威牌名与位置标签，并要求多条追问彼此 distinct。
- `apps/web/src/components/reading/InterpretationView.tsx`
  - follow-up 回答输入状态从按 `prompt` 文本索引改为按 `index` 索引。
  - 初读 `reading_id` 变化时重置 follow-up drafts，避免不同 reading 之间残留输入。
  - 显示追问与回答框时改用稳定 key，不再依赖 prompt 文本唯一性。
- `apps/web/src/server/reading/__tests__/llm-provider.spec.ts`
  - 新增 authority metadata 锁定与 duplicate follow-up 去重回归测试。
  - 新增 authority identity mismatch 直接失败的回归测试。

### 7.3 预期收益
- 即使本地中文模型输出质量一般，用户界面也不会再直接展示 provider 伪造的牌名 / 位置名。
- 重复追问不会再造成两个回答框联动输入。
- 同类问题已被下沉为 provider-level regression，不再只能靠人工复现。
