## 5. M3 最小 LangGraph 接入 (Minimal Migratable Graph)

### 5.1 目标

- 保持 `/api/reading` request / response、`StructuredReading`、history 与前端消费协议不变。
- 不接 LangSmith、checkpoint、streaming、interrupt、人审或外部 LLM。
- 前端只围绕 M3 做契约保护与必要安全摩擦补丁，不重构结果页。

### 5.2 核心改动
- 新增 `apps/web/src/server/reading/graph.ts`，使用 `StateGraph` + `StateSchema` 承载固定节点链路：
  - `classify_question`
  - `hydrate_context`
  - `analyze_intent_friction`
  - `generate_draft`
  - `build_structured_reading`
  - `apply_safety_review`
- `hydrate_context` 继续复用当前 spread/card authority 校验逻辑，并按 spread position 顺序生成 `drawnCards`。
- `analyze_intent_friction` 继续在 Tier 1 hard-stop 时抛 `ReadingServiceError("safety_intercept", ..., 403)`，route 错误映射不变。
- `generate_draft` 继续通过 `getReadingProvider()`，默认仍为 placeholder provider。
- `build_structured_reading` 继续生成 `reading_id`、`locale = "zh-CN"`、`session_capsule = null`，并注入 Tier 2 的 `sober_check` / `presentation_mode = "sober_anchor"`。
- `apply_safety_review` 后再次用 `structuredReadingSchema.parse` 验证最终 payload。
- `apps/web/src/server/reading/service.ts` 缩成薄包装，导出名 `generateStructuredReading` 不变。
- 新增依赖 `@langchain/langgraph` 与 `@langchain/core`，更新 `apps/web/package.json` 与 `package-lock.json`。

### 5.3 UX 与状态保护
- `apps/web/src/components/reading/InterpretationView.tsx` 的 Sober Check 解锁条件改为 `soberInput.trim().length >= 5`。
- Sober Gate 状态迁入 `apps/web/src/context/ReadingContext.tsx`，并按 `reading_id` 隔离，避免 reading 页面 remount 或连续测试流程导致用户输入被清空。
